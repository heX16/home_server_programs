# Техническое задание: Python скрипт для мониторинга трафика

## 1. Цель проекта

Создать легковесный Python-скрипт для подсчета сетевого трафика на интерфейсе `eth0` с сохранением статистики в базу данных SQLite. Скрипт должен работать периодически (раз в час) через systemd timer или cron, не занимая память постоянно.

## 2. Требования

### 2.1 Технические требования
- **Язык:** Python 3
- **База данных:** SQLite
- **Сервер:** Ubuntu 22.04
- **Интерфейс:** eth0
- **Память:** Минимальное использование (скрипт запускается периодически, не в памяти постоянно)
- **Поддержка:** IPv4 и IPv6 трафик

### 2.2 Функциональные требования
- Чтение статистики трафика из `/proc/net/dev`
- Сохранение статистики в SQLite базу данных
- Поддержка интервалов: почасовой, дневной, месячный
- Командная строка для просмотра статистики
- Легковесность (быстрый запуск и завершение)

## 3. Функциональность

### 3.1 Основные функции

#### 3.1.1 Обновление статистики (`update`)
- Читает текущие счетчики из `/proc/net/dev` для интерфейса `eth0`
- Читает последние значения из таблицы `last_snapshot`
- **Проверяет обнуление счетчиков:**
  - Если новое значение < старого → счетчики обнулились
  - В этом случае использует текущее значение как начальную точку (без вычитания)
- **Если счетчики не обнулились:**
  - Вычисляет разницу: `diff = текущее - предыдущее`
- Сохраняет данные в базу данных `traffic`:
  - Timestamp (datetime)
  - Bytes RX (входящий) - разница или текущее значение при обнулении
  - Bytes TX (исходящий) - разница или текущее значение при обнулении
  - Packets RX
  - Packets TX
- Обновляет таблицу `last_snapshot` новыми значениями счетчиков и uptime системы

#### 3.1.2 Просмотр статистики (`show`)
Команды для просмотра:
- **Сводка:** Текущий день, вчера, месяц, всего
- **По часам:** Последние 24 часа (таблица)
- **По дням:** Последние 30 дней (таблица)
- **По месяцам:** Последние 12 месяцев (таблица)
- **Графики:** ASCII-графики трафика по времени
  - График по часам (последние 24 часа)
  - График по дням (последние 30 дней)
  - График по месяцам (последние 12 месяцев)
- **Живой режим:** Текущая скорость (опционально)

## 4. Структура базы данных SQLite

### 4.1 Таблица `traffic`

```sql
CREATE TABLE traffic (
    timestamp DATETIME NOT NULL,
    interface TEXT NOT NULL DEFAULT 'eth0',
    bytes_rx INTEGER NOT NULL,      -- Входящий трафик в байтах
    bytes_tx INTEGER NOT NULL,      -- Исходящий трафик в байтах
    packets_rx INTEGER NOT NULL,     -- Входящие пакеты
    packets_tx INTEGER NOT NULL,    -- Исходящие пакеты
    bytes_total INTEGER NOT NULL,   -- Общий трафик (rx + tx)
    PRIMARY KEY (timestamp, interface)
);

CREATE INDEX idx_timestamp ON traffic(timestamp);
CREATE INDEX idx_interface ON traffic(interface);
```

### 4.2 Таблица `last_snapshot` (для хранения последних считанных значений счетчиков)

```sql
CREATE TABLE last_snapshot (
    interface TEXT PRIMARY KEY,
    last_bytes_rx INTEGER NOT NULL,        -- Последнее значение RX байт из /proc/net/dev
    last_bytes_tx INTEGER NOT NULL,        -- Последнее значение TX байт из /proc/net/dev
    last_packets_rx INTEGER NOT NULL,       -- Последнее значение RX пакетов
    last_packets_tx INTEGER NOT NULL,       -- Последнее значение TX пакетов
    last_timestamp DATETIME NOT NULL,       -- Время последнего чтения счетчиков
    system_uptime_seconds INTEGER           -- Uptime системы на момент чтения (для проверки обнуления)
);
```

**Назначение таблицы `last_snapshot`:**
- Хранит последние считанные значения счетчиков из `/proc/net/dev`
- Используется для вычисления разницы между обновлениями
- Позволяет определить обнуление счетчиков (если новое значение < старого)

## 6. Технические детали

### 6.1 Чтение статистики из /proc/net/dev

Формат строки в `/proc/net/dev`:
```
Inter-|   Receive                                                |  Transmit
 face |bytes    packets errs drop fifo frame compressed multicast|bytes    packets errs drop fifo colls carrier compressed
  eth0: 12345678 12345    0    0    0     0          0         0    98765432 98765    0    0    0     0       0          0
```

Нужно извлечь:
- Bytes RX: 2-е поле (байты получено)
- Packets RX: 3-е поле (пакеты получено)
- Bytes TX: 10-е поле (байты отправлено)
- Packets TX: 11-е поле (пакеты отправлено)

### 6.2 Обработка обнуления счетчиков

Счетчики в `/proc/net/dev` могут обнулиться при:
- Перезагрузке системы
- Перезапуске сетевого интерфейса (`ip link set eth0 down/up`)
- Сбросе счетчиков вручную

**Алгоритм обработки:**

1. **Чтение текущих счетчиков** из `/proc/net/dev` для интерфейса
2. **Чтение последних значений** из таблицы `last_snapshot`
3. **Проверка обнуления:**
   - Если `новое_значение < старое_значение` → счетчики обнулились
   - В этом случае **НЕ вычислять разницу** (будет отрицательное значение)
   - Вместо этого использовать текущее значение как начальную точку
   - Сохранить в `traffic` запись с текущим значением счетчиков (без вычитания)
4. **Если счетчики не обнулились:**
   - Вычислить разницу: `diff = новое - старое`
   - Сохранить разницу в таблицу `traffic`
5. **Обновить `last_snapshot`** новыми значениями счетчиков и текущим временем

**Дополнительная проверка (опционально):**
- Читать uptime системы (`/proc/uptime`) и сохранять в `last_snapshot.system_uptime_seconds`
- Если uptime системы меньше предыдущего → была перезагрузка → счетчики обнулились

**Примечание:**
- В современных ядрах счетчики обычно 64-bit, переполнение крайне маловероятно
- Переполнение выглядело бы как: новое ≈ 0, старое ≈ max_uint64
- Обнуление выглядит как: новое < старое (где новое обычно небольшое значение)

### 6.3 Форматирование единиц измерения

- Bytes → B, KB, MB, GB, TB (десятичные, 1000 байт = 1 KB)
- Human-readable формат: `150.00 MiB`, `2.49 GiB`

## 8. Развертывание

### 8.1 Установка

1. Скопировать скрипты в `/usr/local/bin/traffic_monitor/`
2. Создать директорию для БД: `/var/lib/traffic_monitor/`
3. Установить права: `chmod +x traffic_monitor.py`
4. Настроить systemd timer или cron

### 8.2 Systemd Timer

**Сервис:** `/etc/systemd/system/traffic-monitor.service`
```ini
[Unit]
Description=Update traffic statistics
After=network-online.target

[Service]
Type=oneshot
ExecStart=/usr/bin/python3 /usr/local/bin/traffic_monitor/traffic_monitor.py update
```

**Таймер:** `/etc/systemd/system/traffic-monitor.timer`
```ini
[Unit]
Description=Update traffic statistics every hour

[Timer]
OnCalendar=hourly
Persistent=true

[Install]
WantedBy=timers.target
```

### 8.3 Cron (альтернатива)

```bash
# Обновление каждый час
0 * * * * /usr/bin/python3 /usr/local/bin/traffic_monitor/traffic_monitor.py update
```

### Формат ASCII-графиков:

- **Вертикальная ось (строки):** Время (часы / дни / месяцы) — новые периоды сверху.
- **Горизонтальная ось (столбцы):** Объём трафика (MiB / GiB).
- **Максимальное значение:** Автоматически определяется из данных (максимальный расход за период).
- **Масштаб:** График масштабируется под максимум, сохраняя пропорции.
- **Тип отображения:**
  - Каждый бар показывает **только расход за этот конкретный час/день/месяц**.
  - Нет накопления предыдущих значений.
- **Символы:** Используются Unicode-блоки (`█ ▌`) для плавного отображения уровня.
- **Легенда:**
  - Каждая строка показывает время (слева) и значение трафика (справа).
  - Нижняя шкала — числовой диапазон значений.



### 12.1 Алгоритм построения графика

1. **Получить данные** за период (часы/дни/месяцы)
2. **Определить максимальное значение** трафика за период
3. **Рассчитать ширину графика:**
   - Рекомендуемая ширина: 50-80 символов (или параметр `--width`)
   - Минимальная ширина: 30 символов
4. **Масштабирование:**
   - Максимальное значение = 100% ширины графика
   - Каждое значение масштабируется: `bar_length = (value / max_value) * width`
5. **Построение столбцов:**
   - Использовать Unicode блоки для плавного отображения
   - Символы: `█`  `▌`
   - Если значение < 1 символа, использовать частичный символ
6. **Форматирование:**
   - Время слева (выровнено)
   - График в середине
   - Значение справа (выровнено)
   - Шкала снизу (значения по горизонтали)

### 12.2 Параметры графиков

- `--width`, `-w` - Ширина графика в символах (по умолчанию: 60)
- `--max-height` - Максимальная высота (количество строк, по умолчанию: не ограничено)
