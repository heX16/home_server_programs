
Конфигурация
================

Редактирование
-----------------
Конфигурация это таблица, редактируется через LibreOffice Calc (или Excel).


Описание
-----------

Программа получает и затем передает данные.

Note: В mqtt принято для одной сущности (одной переменной) иметь два пути - путь для чтения и путь для записи.


Данные она получает из путей указанных в колонке **mqtt_switch_read**.
По умолчанию программа реагирует на любые записаные в mqtt_switch_read данные,
но можно указать фильтр чтобы реакция была на определенные данные.
Для этого сразу после пути (в этом столбце) нужно написать "==" и указать данные которые должны содержаться в пакете.

Note: только операция сравнения.

Сразу после **mqtt_switch_read** идет перечень связанных с этим путем путей записи **mqtt_write**.
Таким образом один элемент конфигурации может занимать несколько строк.

После получения данных программа пишет эти данные в путь указанный в колонке **mqtt_write**.
Путей записи может быть много - они записываются множеством строчек.

В колонке **value** указывается "s" - если требуется просто транслировать полученные данные без изменения.
Если требуется записать данные отличные от изначальных данных то в колонке value можно указать константу или формулу.
((формула считается через eval, см процедуру ScenElem.get_value))

В формуле доступно две переменных:
1. s - значение от mqtt_switch_read.
2. v - предыдущее значение полученное от mqtt_read.

В колонке **pause** можно указать паузу которую должна выждать программа перед записью после получения данных.
Время указывается в секундах, можно указывать дробное число (разделитель запятой - ".").
Если во время выдержки времени пришли новые данные по этому пути то текущая запись отменяется, подготавливаются новые записи (если требуется пауза то она начинается с начала).

Для корректной работы формул при старте программы ей требуется получить данные от обьектов.
Для этого используется колонка **mqtt_read** - при старте программа подписывается на эти пути.

В столбце **type** можно указать тип данных, программа будет производить типизацию входящих данных.
((см. процедуру "retype"))

Также есть столбец **desc** в котором может быть произвольный комментарий.

Столбцы
-----------
mqtt_switch_read — путь MQTT из которого получаем данные
mqtt_write — путь куда пишем данные
value — значение которое записывается.
pause — пауза через которую нужно производить запись.
mqtt_read — путь на который нужно подписаться чтобы получить данные.
